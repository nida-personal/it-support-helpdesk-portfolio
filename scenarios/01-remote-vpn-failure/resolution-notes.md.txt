## Resolution Summary

User could not connect to VPN due to Windows update KB5034441 modifying
network adapter settings and removing VPN firewall exceptions. Older
GlobalProtect client 5.1.4 became incompatible. Resolved by remotely
deploying updated client 5.3.2 via MDM and re-adding firewall exception.

---

## Steps That Resolved the Issue

1. Identified Windows update as trigger via version check
2. Provided hotspot workaround to restore user productivity immediately
3. Deployed GlobalProtect 5.3.2 via MDM remotely
4. Confirmed resolution on both home Wi-Fi and hotspot

---

## Knowledge Base Article Created

Title: VPN Error 800 After Windows Update, GlobalProtect Fix
Applies To: Windows 11, GlobalProtect 5.1.x
Solution: Update to GlobalProtect 5.3.2 via MDM or manual installer.
Re-add VPN firewall exception if cleared by Windows update.
